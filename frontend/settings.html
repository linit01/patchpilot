How often PatchPilot automatically runs an Ansible scan against all hosts to check for updates. Each interval triggers a real SSH check — the same as pressing the Refresh Status button. Shorter intervals keep status fresh but add SSH load to your hosts.

⚠️ UTC is selected. Your container clock is UTC, so schedule times will be treated as UTC. If your schedule is set for 01:50 local time (CST = UTC−6), you must either select America/Chicago here or set the schedule time to 07:50 UTC. Mismatched timezone is the most common reason schedules never fire.
Timezone used for auto-patch schedule windows. Schedule times entered in the Schedules page are interpreted in this timezone. The Docker container defaults to UTC — this must be set to your local timezone for schedules to fire at the expected wall-clock time.

Network & Security

The public URL where PatchPilot is reachable (include protocol and domain, no trailing slash). Used for CORS validation and self-referencing links.
Examples: https://patchpilot.BLAH.com  |  https://patchpilot.lan  |  http://192.168.1.50:8080
⚠️ Wildcard (*) is set. Any website can make authenticated requests to this PatchPilot instance. Only use * on a trusted local network or during development. For production set explicit URLs.
Comma-separated list of origins browsers are allowed to send requests from. Use * for open / local access. For multi-user or internet-facing deployments list each URL explicitly (e.g. https://patchpilot.BLAH.com,https://patchpilot.lan). Requires a container restart to take effect.
ℹ️ CORS origin changes are saved to the database and will be applied on the next container restart. Current active origins are set via the ALLOWED_ORIGINS environment variable.

🍎 macOS / App Store

Off by default. mas requires an active GUI login session and App Store sign-in on the target Mac — it will hang silently in a headless SSH context if those conditions are not met. Enable only after confirming mas outdated works interactively on each host.
IDs shown with their known app name where possible. To find any app's ID run: mas search <name> on the Mac — Xcode (497799835) is excluded by default.
Hard timeout per individual app on the remote Mac — uses the timeout binary to kill a hung mas process so the patch run can continue. Default 600 (10 min) per app. Increase for particularly large apps.
Maximum seconds Ansible waits for all App Store updates to finish on a single host. Default 7200 (2 h). The overall PatchPilot runner timeout is set to this value + 5 min.
